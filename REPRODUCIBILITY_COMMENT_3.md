# Reproducibility Guide — Reviewer Comment 3

This package is intended to support reproducibility of the reviewer-comment-related pipeline and experiments. The included sample data are only for demonstrating the input format and execution flow; they are not intended to reproduce the full-scale quantitative results reported in the manuscript.

本补充代码包用于支撑第三条审稿意见相关流程和实验的可复现性。包内样例数据仅用于演示输入格式和运行流程，不用于复现论文中的完整实验数值结果。

---

## 1. Scope — What This Package Addresses

This lightweight release supports **Reviewer Comment 3** responses related to:

- **Network-level hidden-member expansion** via online APPR (Spider / TTR)
- **Seed degradation** experiments (online and offline APPR)
- **APPR coverage** completion for missing importance files
- **Fusion-weight (ω) sensitivity** at account level: `S(v) = ω * S_ML(v) + (1 - ω) * S_rule(v)`
- **Frozen clean main protocol** (`main_clean_no_label_prior`) without CE/BE label prior in detection

The package covers the **account--community--network hierarchy** pipeline, not the full MLAGDet repository.

## 4. Quick Start with Sample Data

### 4.1 Install dependencies

```bash
python -m venv venv
# Windows: venv\Scripts\activate
pip install -r requirements.txt
pip install -r Spider/requirements.txt
```

### 4.2 Stage 1 — account + community (sample)

```bash
mkdir -p sample_out/combined sample_out/transactions
copy examples\comment3_sample\sample_transactions.csv sample_out\transactions\labeled_transactions.csv
mkdir labels
copy examples\comment3_sample\sample_CE.csv labels\CE.csv
copy examples\comment3_sample\sample_BE.csv labels\BE.csv

python money_laundering_detector.py -i sample_out/transactions -o sample_out -t 0.25 --disable_label_prior
```

This demonstrates input/output layout (community JSON/CSV under `sample_out/combined/`). Metrics on 10-row sample data are **not** paper numbers.

For **online** seed degradation (`phase1_seed_degradation_experiment.py`), copy sample importance files into Spider layout:

```bash
mkdir Spider\0xaaaa000000000000000000000000000000000001\importance
copy examples\comment3_sample\sample_importance\0xaaaa000000000000000000000000000000000001.csv Spider\0xaaaa000000000000000000000000000000000001\importance\
```

### 4.3 Offline seed degradation (no API keys)

```bash
python experiment/phase1_offline_appr_seed_degradation.py ^
  --transactions_file examples/comment3_sample/sample_transactions.csv ^
  --communities_dir examples/comment3_sample ^
  --labels_ce examples/comment3_sample/sample_CE.csv ^
  --labels_be examples/comment3_sample/sample_BE.csv ^
  --repeats 2 --top_k 5,10 --removal_ratios 0,0.5 ^
  --min_malicious_seeds 2 --max_communities 1
```

### 4.4 Online APPR (requires credentials)

```bash
copy .env.example .env
# Edit .env with ETH_RPC_URLS and ETHERSCAN_API_KEYS
# Linux/macOS: export $(grep -v '^#' .env | xargs)

copy examples\comment3_sample\sample_community.json Spider\sample_community.json
python run_analysis.py -a Spider/sample_community.json
```

Online APPR **requires** Ethereum RPC endpoint(s) and Etherscan API key(s), configured via environment variables (see Section 8).

## 5. Replacing Sample Data with Full Paper Data

1. Place full `labeled_transactions.csv` under `transactions/`.
2. Place full `CE.csv` and `BE.csv` under `labels/`.
3. Run Stage 1 to produce `results1/combined/*_community_*.json`.
4. Run APPR coverage / seed-degradation scripts with default paths.
5. Train or copy ML models to `Real-CATS-master/models/` (see training script).
6. Run Phase-2 sensitivity with full data paths.

Default paths assumed by scripts:

| Script | Default input |
|--------|---------------|
| `money_laundering_detector.py` | `-i transactions` |
| `phase1_seed_degradation_experiment.py` | `results1/combined`, `labels/`, `Spider/` |
| `phase1_offline_appr_seed_degradation.py` | `transactions/labeled_transactions.csv` |

## 8. APPR Online Expansion — API Requirements

**Yes**, online APPR via `run_analysis.py` → Scrapy `txs.eth.ttr` requires:

| Variable | Purpose |
|----------|---------|
| `ETH_RPC_URLS` | Comma-separated Ethereum JSON-RPC URLs |
| `ETHERSCAN_API_KEYS` | Comma-separated Etherscan API keys |

Optional (non-Ethereum chains, if used):

- `BSCSCAN_API_KEYS`, `POLYGONSCAN_API_KEYS`, `HECOSCAN_API_KEYS`

Configure via `.env` (copy from `.env.example`) or export in shell. **Do not commit real keys.**

`Spider/BlockchainSpider/settings.py` reads keys only from environment variables in this release package.

## 10. Final Pipeline Parameter Table

These are the **canonical defaults** for the paper-facing MLAGDet pipeline. Code, README, and this package use the same values unless a script explicitly overrides them for sensitivity analysis.

| Symbol | Parameter | Default | CLI / code location |
|--------|-----------|---------|---------------------|
| δ | Initial account-level detection threshold | `0.25` | `money_laundering_detector.py -t` |
| δ_re | Expanded-candidate re-verification threshold | `0.37` | `suspicious_account_detector.py -t` |
| ω | Learning-score fusion weight | `0.6` | `--model_weight` (Stage 1) |
| 1−ω | Rule-score fusion weight | `0.4` | `--rule_weight` (Stage 1) |
| α | APPR teleport parameter | `0.15` | `Spider/.../spiders/ttr.py`, `strategies/ttr.py` |
| ε | APPR residual threshold | `1e-3` | `Spider/.../spiders/ttr.py`, `strategies/ttr.py` |
| β | APPR direction bias | `0.8` | `Spider/.../spiders/ttr.py`, `strategies/ttr.py` |
| η | Jaccard similarity threshold (community fusion) | `0.5` | `money_laundering_detector.py --jaccard_threshold` |
| s_min | Minimum community size | `3` | `--min_community_size` |

### Fusion formula (Stage 1)

```
S(v) = ω × S_ML(v) + (1 - ω) × S_rule(v)
     = model_weight × model_score + rule_weight × rule_score
```

Defaults: `model_weight=0.6`, `rule_weight=0.4`.

Stage 3 re-detection uses equal weights (`model_weight=0.5`, `rule_weight=0.5`) unless overridden.

### APPR/TTR defaults

Online expansion (`run_analysis.py` → Scrapy `txs.eth.ttr`) uses **α=0.15, β=0.8, ε=1e-3** when task JSON omits these fields. Sample task file: `examples/comment3_sample/sample_community.json`.

**Legacy settings (not final pipeline defaults):**

| Setting | Value | Where seen |
|---------|-------|------------|
| β = 0.7 | Spider spider kwargs (pre-unification) | Older crawl JSON / dev runs |
| ε = 1e-5 | Strategy class defaults (pre-unification) | Historical experiments |
| β = 0.6, ε = 0.0001 | `Spider/file.json` | One-off task config |

### Experiment-only parameters

| Parameter | Default | Script |
|-----------|---------|--------|
| Seed removal ratios | `0,0.1,…,0.5` | Phase-1 seed degradation |
| Top-K | `20,50,100,200` | Phase-1 scripts |
| Online repeat count | `20` | `phase1_seed_degradation_experiment.py` |
| Offline repeat count | `30` | `phase1_offline_appr_seed_degradation.py` |
| Offline k-hop | `2` | `--k_hop` |
| Offline PageRank α | `0.85` | NetworkX personalized PageRank (≠ TTR α) |
| Phase-2 ω grid | `0.0…1.0` step `0.1` | `phase2_alpha_sensitivity_experiment.py` |
| Phase-2 threshold search | `0.05…0.95` step `0.01` | Phase-2 script |
